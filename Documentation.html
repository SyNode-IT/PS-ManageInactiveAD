5. Scripts de nettoyage AD

5.1 Find-ADInactiveUsers

But : Trouver et gérer les comptes utilisateurs inactifs.

ParamètreDéfautDescription
-SearchScopeAllAll, OnlyInactiveUsers, OnlyServiceAccounts, OnlyNeverLoggedOn, AllExceptServiceAccounts, AllExceptNeverLoggedOn
-DaysInactive90Jours d'inactivité
-ServiceAccountIdentifiersvcPréfixe/suffixe des comptes de service
-QuarantineOU—OU de quarantaine pour les comptes désactivés
-DisableUsers$falseDésactive les comptes inactifs
-DeleteUsers$falseSupprime les comptes inactifs
# Rapport simple
.\Find-ADInactiveUsers.ps1

# Rapport 60 jours dans l'OU Paris
.\Find-ADInactiveUsers.ps1 -DaysInactive 60 -SearchBase "OU=Paris,DC=corp,DC=local"

# Désactiver et déplacer en quarantaine
.\Find-ADInactiveUsers.ps1 -DisableUsers -QuarantineOU "OU=Disabled Users,DC=corp,DC=local"

# Exclure les VIP
.\Find-ADInactiveUsers.ps1 -ExcludeOU "OU=VIP,DC=corp,DC=local"

Colonnes CSV : Username, Name, LastLogonDate, DistinguishedName

5.2 Find-ADInactiveComputers

But : Trouver et gérer les objets ordinateur inactifs.

ParamètreDéfautDescription
-SearchScopeAllAll, OnlyInactiveComputers, OnlyNeverLoggedOn
-DaysInactive90Jours d'inactivité
-QuarantineOU—OU de quarantaine
-DisableObjects$falseDésactive les ordinateurs inactifs
-DeleteObjects$falseSupprime les ordinateurs inactifs
# Rapport des postes inactifs depuis 120 jours
.\Find-ADInactiveComputers.ps1 -DaysInactive 120

# Désactiver et déplacer en quarantaine
.\Find-ADInactiveComputers.ps1 -DisableObjects -QuarantineOU "OU=Disabled Computers,DC=corp,DC=local"

5.3 Find-ADEmptyGroups

But : Trouver et gérer les groupes AD vides.

ParamètreDéfautDescription
-SearchScopeDomaine entierChemin LDAP
-DeleteObjects$falseSupprime les groupes vides
# Rapport
.\Find-ADEmptyGroups.ps1

# Supprimer (simulation)
.\Find-ADEmptyGroups.ps1 -DeleteObjects -WhatIf

5.4 Find-ADEmptyOU

But : Trouver et gérer les OUs vides.

ParamètreDéfautDescription
-SearchScopeDomaine entierChemin LDAP
-DeleteObjects$falseSupprime les OUs vides
Les OUs protégées contre la suppression accidentelle génèreront une erreur.

6. Scripts d'audit sécurité

6.1 Find-ADLockedAccounts

But : Trouver et optionnellement déverrouiller les comptes verrouillés.

ParamètreDéfautDescription
-UnlockAccounts$falseDéverrouille les comptes
# Lister les comptes verrouillés
.\Find-ADLockedAccounts.ps1

# Déverrouiller
.\Find-ADLockedAccounts.ps1 -UnlockAccounts

CSV : Username, Name, LastLogonDate, LockoutTime, LastBadPasswordAttempt, DistinguishedName

6.2 Find-ADPasswordNeverExpires

But : Trouver les comptes avec le flag « Mot de passe n'expire jamais ».

ParamètreDéfautDescription
-IncludeServiceAccounts$falseInclure les comptes de service
-ServiceAccountIdentifiersvcIdentifiant des comptes de service
-RemoveFlag$falseRetire le flag PasswordNeverExpires
# Rapport
.\Find-ADPasswordNeverExpires.ps1

# Retirer le flag (simulation)
.\Find-ADPasswordNeverExpires.ps1 -RemoveFlag -WhatIf

6.3 Find-ADStalePasswords

But : Trouver les comptes dont le mot de passe est ancien.

ParamètreDéfautDescription
-DaysOld180Jours depuis le dernier changement
-ForceChangeAtLogon$falseForce le changement au prochain logon
# Mots de passe de plus d'un an
.\Find-ADStalePasswords.ps1 -DaysOld 365

# Forcer le changement (simulation)
.\Find-ADStalePasswords.ps1 -DaysOld 90 -ForceChangeAtLogon -WhatIf

CSV : Username, Name, PasswordLastSet, PasswordAgeDays, LastLogonDate, Description, DistinguishedName

6.4 Find-ADPrivilegedAccounts

But : Auditer les membres des groupes privilégiés.

ParamètreDéfautDescription
-Groups10 groupes privilégiés par défautGroupes à auditer
-IncludeNested$falseRésolution récursive des groupes imbriqués

Groupes audités par défaut : Domain Admins, Enterprise Admins, Schema Admins, Administrators, Account Operators, Backup Operators, Server Operators, Print Operators, DnsAdmins, Group Policy Creator Owners

# Audit complet avec résolution récursive
.\Find-ADPrivilegedAccounts.ps1 -IncludeNested

# Groupes spécifiques
.\Find-ADPrivilegedAccounts.ps1 -Groups "Domain Admins","IT-Admins"

7. Scripts de maintenance

7.1 Find-ADDisabledInGroups

But : Trouver les comptes désactivés encore membres de groupes.

ParamètreDéfautDescription
-RemoveMemberships$falseRetire les appartenances aux groupes
# Rapport
.\Find-ADDisabledInGroups.ps1

# Retirer les appartenances (simulation)
.\Find-ADDisabledInGroups.ps1 -RemoveMemberships -WhatIf

7.2 Find-ADObsoleteOS

But : Trouver les ordinateurs sous OS obsolètes.

ParamètreDéfautDescription
-ObsoletePatternsXP à Server 2019Patterns d'OS considérés obsolètes
Adapter -ObsoletePatterns selon votre politique de cycle de vie. Si Server 2019 est encore supporté, retirez-le de la liste.
# Rapport par défaut
.\Find-ADObsoleteOS.ps1

# Uniquement Server 2012 et 2016
.\Find-ADObsoleteOS.ps1 -ObsoletePatterns "*Server 2012*","*Server 2016*"

7.3 Find-ADUnlinkedGPO

But : Trouver les GPO orphelines (non liées).

Prérequis : Module GroupPolicy (GPMC / RSAT)
ParamètreDéfautDescription
-ExcludeGPODefault Domain Policy, Default DC PolicyGPOs à exclure
-DeleteObjects$falseSupprime les GPOs non liées
# Rapport
.\Find-ADUnlinkedGPO.ps1

# Supprimer (simulation)
.\Find-ADUnlinkedGPO.ps1 -DeleteObjects -WhatIf

7.4 Find-ADDuplicateSPN

But : Détecter les SPN en doublon (causent des échecs Kerberos).

Ce script est en lecture seule. Correction manuelle avec :

setspn -d HTTP/serveur.corp.local COMPTE_A_CORRIGER

8. Scripts de santé AD

8.1 Test-ADReplicationHealth

But : Vérifier la réplication AD entre tous les DCs.

Statuts possibles : OK, ERROR, UNREACHABLE

# Vérification avec notification email
.\Test-ADReplicationHealth.ps1 -EnableLogging -EmailTo "admin@corp.local" -SmtpServer "smtp.corp.local"

CSV : SourceDC, SourceSite, Partner, LastReplicationSuccess, MinutesSinceReplication, ConsecutiveFailures, Status

8.2 Test-ADFSMORoles

But : Auditer les 5 rôles FSMO et l'état des DCs.

Rôles vérifiés :

  • Schema Master (Forêt)
  • Domain Naming Master (Forêt)
  • PDC Emulator (Domaine)
  • RID Master (Domaine)
  • Infrastructure Master (Domaine)
.\Test-ADFSMORoles.ps1 -EnableLogging

9. Workflow de nettoyage recommandé

Workflow mensuel

Semaine 1 : AUDIT 1. Find-ADPrivilegedAccounts → Vérifier les comptes à privilèges 2. Find-ADPasswordNeverExpires → Identifier les risques 3. Find-ADStalePasswords → Mots de passe anciens 4. Find-ADDuplicateSPN → Intégrité Kerberos 5. Test-ADReplicationHealth → Réplication 6. Test-ADFSMORoles → Rôles FSMO Semaine 2 : NETTOYAGE OBJETS 1. Find-ADInactiveUsers -DisableUsers -QuarantineOU "..." 2. Find-ADInactiveComputers -DisableObjects -QuarantineOU "..." 3. Find-ADDisabledInGroups -RemoveMemberships Semaine 3 : NETTOYAGE STRUCTURE 1. Find-ADEmptyGroups → Groupes vides 2. Find-ADEmptyOU → OUs vides 3. Find-ADUnlinkedGPO → GPOs orphelines 4. Find-ADObsoleteOS → OS obsolètes Semaine 4 : SUPPRESSION (après validation) 1. Find-ADInactiveUsers -SearchBase "OU=Quarantine,..." -DeleteUsers 2. Find-ADInactiveComputers -SearchBase "OU=Quarantine,..." -DeleteObjects 3. Find-ADEmptyGroups -DeleteObjects

Cycle de vie d'un compte utilisateur

Compte Actif | (Inactif > 90 jours) | v Désactiver (-DisableUsers) | Déplacer (-QuarantineOU "OU=Disabled,...") | (Attendre 30 jours) | v Supprimer (-DeleteUsers -SearchBase "OU=Disabled,...")

10. Planification des tâches

Tâche planifiée hebdomadaire

$Action = New-ScheduledTaskAction -Execute 'powershell.exe' `
  -Argument '-NoProfile -ExecutionPolicy Bypass -File "C:\Scripts\Find-ADInactiveUsers.ps1" -EnableLogging -EmailTo "admin@corp.local" -SmtpServer "smtp.corp.local"'

$Trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Monday -At '06:00'

$Settings = New-ScheduledTaskSettingsSet -RunOnlyIfNetworkAvailable -StartWhenAvailable

Register-ScheduledTask -TaskName 'AD-InactiveUsersReport' `
  -Action $Action -Trigger $Trigger -Settings $Settings `
  -User 'DOMAIN\svc-admanagement' -Password 'MotDePasse' `
  -Description 'Rapport hebdomadaire des utilisateurs inactifs'

Protect from Accidental DeletionOU protégéeRetirer la protection dans ADUC
The term 'Get-GPO' is not recognizedModule GroupPolicy absentInstall-WindowsFeature GPMC

Limitations connues

  • LastLogonDate : Basée sur LastLogonTimestamp, répliquée tous les 9-14 jours entre DCs. Pour une valeur précise, interroger LastLogon sur chaque DC.
  • Send-MailMessage : Obsolète dans PowerShell 7+ mais fonctionnelle. Pour le long terme, envisager Microsoft Graph API.
  • gMSA/MSA : Non distingués des comptes classiques. Utiliser -ExcludeOU pour les exclure.

Vérifier les logs

# Lister les fichiers de log
Get-ChildItem C:\tmp\Logs\ -Filter *.log | Sort-Object LastWriteTime -Descending

# Lire le dernier log
Get-Content (Get-ChildItem C:\tmp\Logs\ -Filter *.log | Sort-Object LastWriteTime -Descending | Select-Object -First 1).FullName
